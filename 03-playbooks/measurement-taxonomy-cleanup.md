# Measurement and Taxonomy Cleanup Playbook

## Purpose

Use this when teams cannot confidently connect activity to outcomes.

## Symptoms

- Multiple definitions for the same metric
- Missing or inconsistent source labels
- Similar forms using different fields
- Reports that require manual interpretation every week
- Teams debating numbers instead of decisions

## Cleanup sequence

### 1. Define the entities

Clarify business unit, brand, market, channel, campaign, audience, offer, funnel stage, and outcome.

### 2. Standardize naming

A naming system should be readable by people and usable by systems.

### 3. Align fields

Comparable paths should collect comparable information.

### 4. Document events

Each meaningful action should have a name, definition, owner, and reporting purpose.

### 5. Create QA rhythm

Review missing values, inconsistent labels, duplicates, and source drift on a regular cadence.

## Output

- Naming guide
- Metric dictionary
- Event map
- Required field list
- QA checklist
- Issue log
- Owner map

## Standard

Taxonomy is infrastructure. If the labels are messy, the decision layer will be messy too.
