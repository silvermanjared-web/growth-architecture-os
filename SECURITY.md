# Security Policy

## Supported Versions

Growth Architecture OS is currently maintained from the `main` branch.

Because this repository is an open-source architecture, documentation, and operating-system framework, security support applies to the current published version of the repository unless otherwise noted.

| Version / Branch | Supported |
| --- | --- |
| `main` | ✅ Yes |
| Previous snapshots, forks, or local copies | ❌ No |

## Reporting a Vulnerability

If you discover a security issue, please do **not** open a public GitHub issue or discussion.

Instead, report the concern privately by using GitHub’s private vulnerability reporting feature, if enabled, or by contacting the repository maintainer directly.

When reporting a vulnerability, please include:

- A clear description of the issue
- The affected file, workflow, dependency, or configuration
- Steps to reproduce, if applicable
- Any relevant screenshots, logs, or proof of concept
- Your recommended fix, if you have one

## What Counts as a Security Issue

Security issues may include:

- Exposed credentials, API keys, tokens, or secrets
- Unsafe configuration examples
- Dependency vulnerabilities
- Instructions that could lead to insecure implementation
- Sensitive information accidentally committed to the repository
- Access control, authentication, or authorization risks in example workflows

General bugs, documentation gaps, or feature requests should be opened as standard GitHub issues instead.

## Response Expectations

Security reports will be reviewed as quickly as reasonably possible.

Expected process:

1. The maintainer reviews the report.
2. If valid, the issue is prioritized based on severity and potential impact.
3. A fix, mitigation, or documentation update is prepared.
4. The reporter may be credited if appropriate and requested.

This project is maintained in good faith, but response times may vary depending on maintainer availability.

## Responsible Disclosure

Please give the maintainer a reasonable opportunity to investigate and address the issue before publicly disclosing it.

Do not exploit the vulnerability, access data that does not belong to you, or perform testing that could disrupt systems, users, or services.

## Scope

This security policy applies only to this repository and its contents.

It does not apply to third-party tools, platforms, vendors, or services referenced by the project. Please report vulnerabilities in those systems through their official security channels.
