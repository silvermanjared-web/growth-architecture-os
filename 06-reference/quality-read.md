# Quality Read

## Purpose

Use this when surface-level performance looks good but the business result is unclear.

## Questions

- Is the activity moving toward the actual outcome?
- Which sources create real progress?
- Which sources create noise?
- Are we comparing like with like?
- Are definitions consistent across the system?

## Output

A quality read should end with one of four calls:

1. Scale
2. Hold
3. Fix the path
4. Recheck the signal

## Standard

Do not reward activity that does not create progress.
