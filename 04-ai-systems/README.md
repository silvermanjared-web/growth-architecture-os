# AI Systems

This section defines how AI fits into the Growth Architecture OS.

The purpose is not to present AI as a novelty layer. The purpose is to show how AI can reduce repeated cognitive load, improve workflow quality, and support better marketing operations without replacing human judgment.

## Files

- `ai-operating-model.md` — principles for practical AI adoption in growth and marketing operations
- `agent-workflows.md` — reusable workflow patterns for briefs, readouts, CRO roadmaps, case studies, and QA
- `prompt-library.md` — source-aware prompts for executive communication and operating-model work
- `governance-and-risk.md` — boundaries for accuracy, confidentiality, tone, ownership, and adoption

## Core point of view

AI is useful when it improves how work gets done.

That means:

- faster first drafts
- cleaner summaries
- better QA
- more reusable playbooks
- stronger source discipline
- less repetitive manual work

It does not mean outsourcing judgment.

## How to use this section

Start with `ai-operating-model.md` for the principles, then use `agent-workflows.md` and `prompt-library.md` to turn repeated work into practical workflows.

Use `governance-and-risk.md` whenever the work involves confidential information, public-facing claims, sensitive communication, or executive decision support.

## Standard

AI should make the operating system lighter, clearer, and more useful. It should not create polished noise.
